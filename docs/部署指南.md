# MAD Biomech Tool 部署指南（本地 + 线上）

## 1. 项目结构
- 后端：`backend/`（FastAPI）
- 前端：`frontend/`（Vite + React）
- 数据：`backend/data/关节盘及牙齿应力数据.xlsx`

---

## 2. 本地开发部署

### 2.1 启动后端
```bash
cd backend
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

后端健康检查：
```bash
curl http://localhost:8000/
curl http://localhost:8000/api/study/meta
```

### 2.2 启动前端
```bash
cd frontend
npm install
```

创建 `.env.local`：
```bash
VITE_API_BASE_URL=http://localhost:8000/api
```

运行：
```bash
npm run dev -- --host 0.0.0.0 --port 4173
```

访问：`http://localhost:4173`

## 3. 线上部署（建议方案）

## 3.1 后端（容器或云主机）
- 用 `uvicorn`/`gunicorn+uvicorn` 启动 FastAPI。
- 通过 Nginx 反向代理，将 `/api` 转发到后端服务。

示例（仅示意）：
```nginx
location /api/ {
  proxy_pass http://127.0.0.1:8000/api/;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
}
```

## 3.2 前端（静态托管）
生产构建前设置：
```bash
VITE_API_BASE_URL=/api
npm run build
```
将 `frontend/dist` 交给 Nginx 或静态托管服务。

## 4. 常见问题排查

### 4.1 右上角一直显示“正在读取元信息...”
一般是前端访问 `/api/study/meta` 失败：

1. 打开浏览器开发者工具 Network，检查 `/api/study/meta`：
   - 404：前端代理或 Nginx 路由错误；
   - 502/504：后端未启动或崩溃；
   - CORS 错误：跨域配置不匹配（本项目默认已放开 CORS）。
2. 在后端机器直接 curl：
```bash
curl http://127.0.0.1:8000/api/study/meta
```
3. 检查前端环境变量 `VITE_API_BASE_URL` 是否正确。

### 4.2 点击“更新结果”后图表不显示
1. 检查 `/api/study/analyze` 是否返回 200；
2. 检查后端是否能读取数据文件 `backend/data/关节盘及牙齿应力数据.xlsx`；
3. 检查 Python 依赖是否齐全（`numpy/pandas/scikit-learn/openpyxl`）。

### 4.3 模型区域白屏
1. 检查 `frontend/public/models/` 下 `glb` 与 `manifest.json` 是否完整；
2. 检查浏览器 console 是否有模型加载 404。
